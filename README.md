# 시각 장애인을 위한 AI 보조 안경 시스템

## 1. 프로젝트 개요

본 프로젝트는 **라즈베리 파이 기반 온디바이스(On-device) 비전 AI 기술**을 활용하여 시각 장애인의 보행을 보조하는 **저비용 경량 스마트 안경 시스템**이다.

인터넷 연결 없이도 **실시간 객체 인식 및 음성 안내**가 가능하도록 설계되었으며, 실제 일상생활에서 착용 가능한 **웨어러블 형태**로 구현하는 것을 목표로 한다.

### 핵심 특징

* **온디바이스 AI 처리** (인터넷 불필요)
* **실시간 사물 인식 및 음성 안내**
* **저전력 / 경량 웨어러블 설계**
* **저비용 구현 (약 10만 원 수준)**

---

# 2. 시스템 구조

## 2.1 전체 시스템 구성

시스템은 다음과 같은 구조로 동작한다.

```
카메라 → 라즈베리파이(AI 객체 인식)
            ↓
      상황 분석
            ↓
       음성 안내 생성
            ↓
USB DAC → 앰프 → 골전도 진동자
```

즉,

**영상 인식 → 상황 판단 → 음성 안내 → 골전도 전달**

과정을 통해 시각 정보를 청각 정보로 변환한다.

---

# 3. 웨어러블 설계

## 3.1 배선 구조 (Wire Management)

착용자의 편의성을 위해 배선을 의복 내부로 숨기는 **Invisible Wiring 구조**를 적용하였다.

### 배선 경로

**① 안경부**

* 카메라
* 골전도 진동자

→ 안경 다리를 따라 배선 정리
→ 목 뒤 방향으로 케이블 이동

**② 상체**

* 셔츠/티셔츠 내부로 케이블 통과
* 어깨 → 등 → 허리 방향 이동

**③ 본체**

* 허리 파우치에

  * 라즈베리 파이
  * 보조배터리

최종 연결

### 사용자 경험

이 구조는 **넥밴드형 이어폰과 유사한 착용감**을 제공하며
외부에서 **케이블이 거의 보이지 않는 구조**를 만든다.

---

# 4. 열 관리 및 하드웨어 보호

라즈베리 파이의 안정적인 작동을 위해 **발열 관리 시스템**을 적용하였다.

## 4.1 방열 설계

### ① CPU 방열판

* 구리 또는 알루미늄 방열판
* CPU 열 분산

### ② 금속 케이스

* 알루미늄 케이스 사용
* 케이스 자체를 **대형 히트싱크**로 활용

### ③ 파우치 통풍 구조

파우치에 **통기구(Air-hole)**를 추가하여

* 공기 순환 확보
* 팬 없이도 발열 제어

---

# 5. AI 기반 주요 기능

## 5.1 실시간 객체 인식

AI 모델(YOLO 등)을 이용하여 다음 객체를 인식한다.

* 사람
* 차량
* 의자
* 문
* 계단
* 장애물

사용자의 이동 경로에 위험한 물체만 음성으로 안내한다.

---

## 5.2 거리 추정 및 위험 경고

객체가 화면에서 차지하는 **픽셀 비율**을 이용하여 거리를 추정한다.

예시

| 거리    | 경고 방식  |
| ----- | ------ |
| 3m 이상 | 정보 안내  |
| 1~3m  | 음성 경고  |
| 1m 이하 | 강한 경고음 |

---

## 5.3 텍스트 읽기 기능 (OCR)

버스 번호, 표지판 등 텍스트를 읽어 음성으로 안내한다.

사용 기술

* Tesseract
* EasyOCR

예시

```
"버스 번호 720번입니다"
"남자 화장실입니다"
```

---

## 5.4 환경 상태 인식

단순 객체 인식을 넘어 **상태 정보**를 제공한다.

예

* 열린 문 / 닫힌 문
* 올라가는 계단 / 내려가는 계단

이를 통해 **보행 판단을 쉽게 만든다.**

---

## 5.5 낙상 감지

카메라 영상의

* 급격한 흔들림
* 비정상적인 자세

를 감지하면 **자동 응답 모드**가 활성화된다.

---

# 6. 초음파 센서를 활용한 충돌 방지

## 기능

카메라의 한계를 보완하기 위해 **초음파 센서**를 사용한다.

### 역할

* 30cm 이내 장애물 감지
* 야간 또는 사각지대 보완

### 장점

* 연산량 거의 없음
* 전력 소모 매우 적음
* 즉각적인 경고 가능

---

# 7. 오디오 시스템 구조

소리는 다음 과정을 통해 생성된다.

### 데이터 흐름

```
파이썬 코드
   ↓
USB DAC (디지털 → 아날로그 변환)
   ↓
앰프 (신호 증폭)
   ↓
골전도 진동자
   ↓
사용자 청각 전달
```

### 장점

* 주변 소리 차단 없음
* 귀를 막지 않음
* 안전한 보행 가능

---

# 8. 하드웨어 연결 방식

납땜 없이 연결 가능하도록 **모듈형 구조**를 사용하였다.

### 주요 연결

| 장치      | 연결 방식          |
| ------- | -------------- |
| 카메라     | 라즈베리 파이 카메라 포트 |
| 초음파 센서  | GPIO           |
| 오디오     | USB DAC        |
| 골전도 스피커 | DAC 출력         |

---

# 9. 안경 모듈 제작 방법

## 9.1 열수축 튜브 방식

카메라를 안경 다리에 고정하기 위해 **열수축 튜브**를 사용한다.

### 방법

1. 카메라를 안경 다리에 올림
2. 열수축 튜브로 덮음
3. 드라이기로 가열

### 효과

* 단단한 고정
* 깔끔한 외형
* 전문 장비 같은 마감

---

## 9.2 탈부착 구조

카메라 모듈을 **자석 + 가이드 핀** 방식으로 설계한다.

### 구조

1. 자석 → 1차 고정
2. 가이드 핀 → 위치 정렬

### 장점

* 카메라 방향 고정
* 모듈 교체 가능

# 12. 기술적 차별성

## 12.1 지능형 정보 필터링

기존 시스템 문제

> 모든 객체를 음성으로 안내 → 정보 과잉

본 시스템

> 사용자의 **이동 경로와 겹치는 물체만 안내**

예

* 옆에 의자 → 무시
* 앞에 의자 → 안내

이를 **인지 부하 최소화 알고리즘**이라 한다.

---

## 12.2 센서 융합 (Sensor Fusion)

카메라와 거리 센서를 함께 사용한다.

예

* 카메라: 사람 인식
* 센서: 거리 측정

두 데이터를 교차 검증하여 **오인식 감소**.

---

## 12.3 상황 기반 정보 제공

실내 / 실외 환경에 따라 안내 정보를 변경한다.

### 실외

* 차량
* 신호등
* 계단

### 실내

* 문
* 의자
* 화장실

이를 **Context-aware 안내**라고 한다.

### 부착 위치

* 안경

사용자 취향에 따라 선택 가능하다.

# 14. 프라이버시 보호
